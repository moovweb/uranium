Overview

Introduction to Uranium

Uranium offers web page authors the ability to add several convenient Javascript interactions to the HTML elements on their pages, without writing a single line of Javascript.

For example, you might have a page with a set of sub-sections that each have headers followed by related content, one right after the other. With the Uranium toggler interaction, you could transform that into an accordion just by adding a few data attributes to the existing HTML elements.

Uranium interactions are not opinionated about styling and layout. You should be able to add the Uranium script and a few data attributes to practically any working web page that uses jQuery, and take advantage of these useful interactions right where you need them most.

Basic Requirements

Since Uranium builds on jQuery, the minimum you need to do to get Uranium working on your page is to include the jQuery script followed by the Uranium script. A good place to do this is right before the closing tag for the body element on your HTML document.

Data Attributes

Data attributes are extra properties that can be added to any HTML element. We use data attributes instead of id or class properties of HTML elements so that Uranium can be injected into existing pages as easily as it can be added to new pages, without affecting existing styles and Javascript.

They syntax is simple. Just add the data attributes the same way you would add any other properties to an HTML element. For example, to tell Uranium that a specific H2 element should be used as a toggle button to show and hide the following paragraph, you might start with something like this:

Interaction Sets and Components

Every interaction has at its core a set and a series of components. The set is unique for every interaction: data-ur-set="toggler" for the toggler interaction, etc. The set attribute should be used to wrap the whole interaction.

There are usually a few components per interaction. Some may be required to get the interaction to work, but most are usually optional. For example, a toggler has a button and content, and at least one of each is required if you want the interaction to work. Components are designated in the following way: they'll have data-ur-{interaction}-component="thing".

Most of the interactions also have some extra properties. For example, carousel has an optional data-ur-carousel-component="count", which is used to display the count of the currently selected item in a carousel.

Recommended Styles

Uranium tries to be as unopinionated as possible when it comes to styling. Uraniun interactions typically perform their magic by adding, removing, or changing the values associated with data attributes on HTML elements. How you choose to style these different states is entirely up to you.

That said, there are some basic types of styles that will be needed to see how the Uranium interactions behave. Generally, you are going to want the toggler interaction to toggle whether or not that toggler's content can be seen. One way to do that—although by no means the only way—is to have the CSS apply a rule of display:none to toggler content when it is supposed to be seen, and a rule of display:block when it is not supposed to be seen.

For all of our interactions, we provide a set of recommended styles that will make the behavior visible, but those are for demonstration purposes only. Uranium can be used with any CSS you can dream up!
