Swipe Toggle

core functionality

swipe toggle is a uranium attribute management system, by hitting the next button or swiping it will automatically toggle the data-ur-state to active, prev or next where appropriate. using the css square bracket selector you can key off of these changes and style it to your needs. there is absolutely no assumed style or required tags other than the setting your slider object with the data-ur-swipe-toggle-component="slider", data-ur-set="swipe_toggle" and a data-ur-id="some_id"

auto population

if you wish to auto populate the carousel at some time after initialization, you can do so using the autoPopulate(obj, location) function. you must feed it an array or object with valid formed html or text and a "top" or "bottom" argument to prepend or append your html.

preferences

every swipe toggle element has pre set preferences that can be changed at any point.
Code Example
Ur.Widgets.SwipeToggle.your_widget.preferences = { dots: false, axis: "x", swipeUpdate: true, sensitivity: 10, loop: true, touchbuffer: 20, tapActive: false, touch: true, jump: 1, loop: true, autoSpeed: 500 };
  • axis: ("x", "y") based on expected swipe direction for attribute change
  • swipeUpdate: (bool) allows single swipe mode
  • sensitivity: (num) contains a pixel distance before triggering multi swipe attribute change
  • touchbuffer: (num) the distance swiped before triggering an update
  • tapActive: (bool) allow tapping to activate swipe element.
  • touch: (bool) allow touch movement
  • jump: (num) default movement number usually 1
  • loop: (bool) allow looping
  • autoSpeed: (num) auto scrolling timer in milli-seconds

flags

some flags are set when auto scrolling takes place.
  • touched: (bool) must be reset to allow autscrolling after first touch event on carousel
  • autoID: (num) the numeric value/name of the setInterval() for auto scrolling

events

to assist you with adding modefications to this widget there are 3 events that are fired on the slider element only.
Code Example
x$(".your_element").on("update" function () { console.log("widget updated") });
  • loaded: this is fired after the object is created but just prior to being placed in the Ur.Widgets object
  • preferences: this is called directly after placing the swipe toggle object into the Ur.Widgets object and is the best time to change your preferences.
  • update: fired every time the attributes have changed through this widget.
