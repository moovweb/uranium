When a user types incorrect information into the input field, on clicking outside of the input, an error attribute will be assigned to the input. Also, optionally, an error message will appear explaining the error.

basic_tab: data-ur-set="validator": add this attribute to a div that wraps a single input data-ur-validator-component="*": add this attribute to the input in the set; possible values are email, ccnum, required data-ur-validator-component="error": add this optional attribute after the input; it will data-ur-validator-minval / data-ur-validator-maxval: can be used to specify a minimum or maximum number for the input data-ur-validator-ccard-type="*": attribute is added to the credit card input, giving the type of credit card entered advanced_tab: Required: email/ccnum/required: - add this attribute to the input node Optional: Error: - add this attribute to a div after the input, but within the set ---

Demonstration

In the example below, the first input cannot be blank. It does not have an error component so no message shows up. The next two examples require a valid e-mail and credit card number respectively. They do have the error component, so error messages appear. The final input must have a value between 3 and 16.

If necessary, multiple components can be added to an input. For example, if an input needs to be filled in and a valid e-mail, the attribute on the input would be data-ur-validator-component="email required".

Currently, the validator can only apply to input. We are working on making the numerical interaction options applicable to select, also!

Standards

Uranium uses regular expressions to validate certain information. If you are interested in what in particular is used, they are detailed below.

  • email: must have an @ symbol followed by a period
  • ccnum: using the Luhn algorithm, Uranium determines if a credit card is real or not; then using regular expressions figures out if the credit card is a Visa, MasterCard, American Express, or Discover Card.

We are working on making these options customizable.
